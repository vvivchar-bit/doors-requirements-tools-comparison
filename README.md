# Порівняльна характеристика IBM DOORS та DOORS Web Access

## Тема роботи

Порівняльна характеристика основних інструментів щодо управління вимогами **Dynamic Object Oriented Requirements System** для Linux, Windows, Solaris та веб-клієнта **DOORS Web Access**.

## Мета роботи

Здійснити порівняльну характеристику інструментів IBM DOORS та DOORS Web Access і визначити доцільність їх використання для управління вимогами у межах навчального проєкту **«Програмна система “Готель”»**.

## Проєкт

**Програмна система «Готель»** — навчальний вебзастосунок для перегляду номерів, пошуку доступних варіантів, створення бронювань, збереження даних клієнтів та контролю стану номерів.

У межах цього проєкту інструменти управління вимогами можуть використовуватися для:

- опису функціональних вимог;
- опису нефункціональних вимог;
- трасування вимог;
- контролю змін;
- зв’язку вимог із тестами;
- погодження вимог із замовником або викладачем.

---

## Основні інструменти порівняння

У роботі розглянуто такі варіанти використання IBM DOORS:

| Інструмент / платформа | Коротка характеристика |
|---|---|
| **DOORS Client для Windows** | Повнофункціональний клієнт для створення, редагування, структурування і трасування вимог |
| **DOORS для Linux** | Доцільний переважно для серверної інфраструктури та розгортання компонентів |
| **DOORS для Solaris** | Історична платформа, яка в актуальній версії DOORS 9.7.2.10 не підтримується |
| **DOORS Web Access** | Веб-клієнт для доступу до вимог через браузер |

## Застосування до проєкту «Готель»

Для проєкту **«Програмна система “Готель”»** IBM DOORS може використовуватися для управління такими групами вимог:

| Категорія вимог | Приклади |
|---|---|
| Функціональні вимоги | Перегляд номерів, пошук номерів, створення бронювання |
| Нефункціональні вимоги | Зручність інтерфейсу, швидкість роботи, надійність збереження даних |
| Вимоги до даних | Ім’я клієнта, телефон, email, номер кімнати, дата заїзду та виїзду |
| Вимоги до інтерфейсу | Статистика готелю, список номерів, форма бронювання |
| Вимоги до тестування | Перевірка пошуку, бронювання, LocalStorage, відсутності помилок у консолі |

---

## Узагальнений висновок

Для навчального проєкту **«Програмна система “Готель”»** найбільш доцільним є використання:

- **DOORS Client для Windows** — для повного створення, редагування, структурування і трасування вимог;
- **DOORS Web Access** — для перегляду вимог, погодження змін і спільної роботи через браузер.

Linux доцільно розглядати як платформу для серверної інфраструктури, а Solaris — лише як історичний пункт порівняння, оскільки в актуальній версії DOORS 9.7.2.10 та DOORS Web Access 9.7.2.10 Solaris не підтримується.
