# Порівняльна характеристика IBM DOORS та DOORS Web Access

## Мета порівняння

Метою порівняння є визначення особливостей використання IBM Engineering Requirements Management DOORS та DOORS Web Access для управління вимогами у межах навчального проєкту «Програмна система “Готель”».

IBM DOORS використовується для управління вимогами, трасування зв’язків між вимогами, контролю змін, аналізу вимог та організації роботи над складними програмними системами.

## Порівняльна таблиця

| Інструмент / платформа | Призначення | Переваги | Обмеження | Доцільність для проєкту «Готель» |
|---|---|---|---|---|
| **DOORS Client для Windows** | Повноцінний клієнт для створення, редагування, структурування та трасування вимог | Повний функціонал DOORS; підтримка модулів вимог; трасування; робота з атрибутами; можливість використання DXL | Потребує встановлення клієнта; залежить від Windows-середовища; складніший для початківців | Найкращий варіант для повного адміністрування вимог проєкту |
| **DOORS для Linux** | Використання Linux у серверній інфраструктурі DOORS / DWA | Підходить для серверного розміщення; стабільність; зручність для корпоративної інфраструктури | Не є основним настільним клієнтом DOORS у сучасній версії; потребує адміністрування сервера | Доцільний для серверної частини, але не обов’язковий для навчального проєкту |
| **DOORS для Solaris** | Історична платформа для розгортання окремих компонентів DOORS | Може розглядатися для порівняння старих корпоративних середовищ | В актуальній версії DOORS 9.7.2.10 та DOORS Web Access 9.7.2.10 Solaris не підтримується | Для проєкту «Готель» недоцільний |
| **DOORS Web Access** | Веб-клієнт для доступу до вимог через браузер | Не потребує встановлення настільного клієнта; зручний для перегляду вимог; підходить для участі замовника, викладача або членів команди | Має менше можливостей порівняно з повним DOORS Client; залежить від серверної частини | Доцільний для перегляду вимог, погодження та спільної роботи |

## Застосування до проєкту «Програмна система “Готель”»

У межах проєкту «Програмна система “Готель”» IBM DOORS може використовуватися для управління такими вимогами:

| Категорія вимог | Приклади для системи «Готель» |
|---|---|
| Функціональні вимоги | Перегляд номерів, пошук номерів, створення бронювання, скасування бронювання |
| Нефункціональні вимоги | Зручність інтерфейсу, швидкість роботи, надійність збереження даних |
| Вимоги до даних | Дані клієнта, номер телефону, email, номер кімнати, дата заїзду та виїзду |
| Вимоги до інтерфейсу | Блок статистики, список номерів, форма бронювання, список бронювань |
| Вимоги до тестування | Перевірка пошуку, бронювання, LocalStorage, відсутності помилок у консолі |

## Висновок

Для навчального проєкту «Програмна система “Готель”» найбільш доцільним є поєднання DOORS Client для Windows та DOORS Web Access.

DOORS Client для Windows доцільно використовувати для повного створення, редагування, структурування і трасування вимог. DOORS Web Access зручний для перегляду вимог, погодження змін та участі користувачів, які не працюють безпосередньо з повним клієнтом DOORS.

Linux доцільно розглядати як платформу для серверної частини, а Solaris — лише як історичний пункт порівняння, оскільки в актуальній версії DOORS 9.7.2.10 та DOORS Web Access 9.7.2.10 Solaris не підтримується.
